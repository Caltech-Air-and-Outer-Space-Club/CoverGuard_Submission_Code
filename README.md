## What Is CoverGuard?
CoverGuard is a 4-satellite SmallSat constellation that diagnoses cover-crop failure **from orbit** — not just detecting that vegetation is stressed, but identifying the specific root cause: moisture stress, nutrient deficit, poor establishment, excess wetness, or pest and disease pressure.

Existing satellite tools flag low vegetation. CoverGuard explains *why* it's low — 10 to 14 days before symptoms are visible to any ground scout or sensor, with 96% accuracy, delivered to the farmer within 95 minutes of imaging.

The system is built entirely on two freely available NASA datasets:
- **OPERA DIST-ANN-HLS** — annual land surface disturbance layers at 30 m
- **HLS Landsat (HLSL30)** — harmonized surface reflectance at 30 m

A 5 MB LightGBM classifier runs entirely onboard a low-power CubeSat, reducing transmitted data by 99.96% — from 128 MB of raw imagery to ~50 bytes per parcel. At $0.75/acre/season, CoverGuard targets a $30 million U.S. precision-agriculture market and provides the first orbit-derived verification layer for USDA conservation compliance and carbon credit programs.

## File Descriptions

### `main.py`
Top-level orchestrator. Runs the full pipeline in paper section order by calling each script as a subprocess. Accepts `--section 2`, `--section 3`, or `--section 4` to run a single section. Prints a description of each section before it runs.

---

### `ground_pipeline_section2.py` — §2: Ground Data Processing Pipeline
Downloads and fuses two NASA datasets for a California Central Valley AOI (~30 km × 30 km):

- **OPERA DIST-ANN-HLS**: annual disturbance status, confidence, anomaly magnitude, and duration per 30 m pixel
- **HLS Landsat (HLSL30)**: surface reflectance bands for NDVI and EVI computation

The pipeline applies cloud masking (Fmask), reprojects OPERA onto the HLS grid, segments the scene into a ~300-cell grid, and computes zonal statistics per cell. Each cell receives a **Field Health Score (1–10)** — a weighted composite of NDVI level (35%), anomaly deviation (20%), disturbance severity (25%), within-cell variability (10%), and valid pixel coverage (10%).

Outputs a GeoJSON, two CSVs, and a six-panel diagnostic figure.

> Requires NASA Earthdata account and internet access.

---

### `onboard_classifier_section3.py` — §3: Onboard Failure-Cause Classifier
Trains and evaluates the LightGBM classifier that runs onboard the CoverGuard satellite. Takes 6 raw multispectral band reflectances (Blue, Green, Red, Red-Edge, NIR, SWIR1), computes 8 spectral indices, and classifies each pixel into one of six failure classes:

| Class | Description |
|---|---|
| `HEALTHY` | No active failure mode |
| `MOISTURE_STRESS` | Drought / insufficient soil water |
| `EXCESS_WETNESS` | Ponding / waterlogging / saturated root zone |
| `NUTRIENT_DEFICIT` | Low nitrogen, chlorophyll, or organic matter |
| `POOR_ESTABLISHMENT` | Sparse or patchy canopy in early growth stages |
| `PEST_OR_DISEASE` | Biotic stress or pest hotspot |

Trained on 297,019 labeled pixels with per-class Gaussian spectral augmentation for rare classes. Labels are derived from research-backed agronomic thresholds (Gao 1996, Schepers 1996, Setter & Waters 2003). Achieves **macro-F1 = 0.96** at **<1 ms/pixel** inference — deployable on ARM hardware in a 5 MB model file.

Includes a live demo running 6 hand-crafted spectral scenarios (drought, waterlogging, nutrient deficit, poor establishment, pest, healthy) through the trained model.

> Fully offline. No NASA account required.

---

### `downlink_simulation_section4.py` — §4: Adaptive Downlink Scheduler
Orchestrates four progressive downlink scheduler simulations. Runs each stage as an isolated subprocess to avoid class-name conflicts between implementations. Each stage builds on the previous:

| Stage | File | What it adds |
|---|---|---|
| 1 | `stage1_basic_scheduler.py` | Baseline: indivisible packets, naive FIFO vs adaptive 0/1 knapsack |
| 2 | `stage2_fragmentation.py` | Resumable raw-image fragmentation across multiple passes |
| 3 | `stage3_lookahead_deadlines.py` | Deadline-aware lookahead — urgency multipliers based on time-to-expiry |
| 4 | `stage4_adaptive_scheduler.py` | Full system: value classes, stale fault suppression, constellation duplicate removal |

Each stage runs a single-seed diagnostic followed by a 30-seed Monte Carlo simulation. The adaptive scheduler (Stage 4) improves median utility **20.8%** over naive FIFO across all seeds. Full results are saved to `outputs/section4_downlink_simulation/`.

> Fully offline. No NASA account required.

---

### `classifier_training_datasets/`
Training data for the onboard classifier. `agriculture_dataset.csv` is the primary dataset (297k synthetic rows with NDVI, SAVI, and chlorophyll content). `merged_training_data.csv` combines synthetic data with real HLS-derived spectral measurements pulled via `hls_data_pull.py`. Missing spectral indices (NDMI, NDRE, EVI, NDWI, MSAVI) are augmented from per-class Gaussian profiles grounded in remote sensing literature.

### `trained_model/`
Serialized LightGBM booster generated by Section 3. Reloaded automatically when running `--demo-only`. The `.txt` format is the LightGBM native text format — human-readable and ARM-deployable.

### `downlink_stages/`
Internal implementation files for the four downlink scheduler stages. Not intended to be run directly — called by `downlink_simulation_section4.py`. Each file is self-contained and defines the same class names (`SimConfig`, `Packet`, `Contact`, etc.) with progressively more capable implementations.
